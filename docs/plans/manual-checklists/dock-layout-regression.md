# Dock Layout Regression Checklist

本清单用于人工回归验证 **Vibe Cat 的 Dock 相关布局逻辑** 在多屏幕、不同缩放、Dock 自动隐藏、Dock 位于左/右侧等场景下的表现是否符合预期。

支持边界（本轮应满足）：

- 多屏支持：猫会锚定到 **Dock 所在屏幕**，而不是固定主屏。
- 自动隐藏 Dock：猫会跟随 Dock **显示/隐藏** 重新贴边（避免漂浮在错误的基线）。
- 左/右 Dock：当前版本采用 **floor mode**，猫沿锚定屏幕 `visibleFrame` 的底边活动，**不实现竖向贴边**。

不在本清单覆盖范围：

- 动画状态/行为映射（走路、奔跑、睡觉等）本身的正确性。
- 气泡/权限提示/子代理迷你猫的业务逻辑。

---

## 预置条件

- macOS 系统可进入“系统设置 / 桌面与 Dock（Desktop & Dock）”。
- 已启动应用（任选其一）：
  - 运行已构建的 `VibeCat.app`
  - 或在仓库根目录执行 `cargo run -- cat` 启动猫窗口
- 若涉及双屏：
  - 可在“系统设置 / 显示器（Displays）”调整屏幕排列（左右位置）与缩放（2x/1x）。

---

## 回归项（逐条勾选）

### 1) 单屏，Dock 在底部，自动隐藏关闭

- [ ] 设置：Dock 位置 = 底部；“自动隐藏和显示 Dock” = 关闭
- [ ] 期望：猫贴在 Dock 上方，基线与 Dock 顶边对齐（无明显漂浮/穿插）
- [ ] 期望：猫的水平活动范围限制在该屏幕可见区域内（不越界到屏幕外）

### 2) 单屏，Dock 在底部，自动隐藏开启/关闭切换

- [ ] 初始设置：Dock 位置 = 底部；“自动隐藏和显示 Dock” = 关闭；确认猫已贴边
- [ ] 切换：打开“自动隐藏和显示 Dock”，将鼠标移开 Dock 触发隐藏
- [ ] 期望：Dock 隐藏后，猫重新贴到新的可见底边（不悬空在原 Dock 基线）
- [ ] 切换：将鼠标移到底部呼出 Dock（或触发显示），再关闭自动隐藏
- [ ] 期望：Dock 显示/关闭自动隐藏后，猫再次重新贴边且无明显跳屏/错位

### 3) 双屏，主屏在左，Dock 在右侧屏幕底部

- [ ] 设置显示器排列：主屏在左，副屏在右（右侧屏幕全局 x 为正）
- [ ] 将 Dock 移动/切换到右侧屏幕底部（常见做法：在右屏底部“顶住”鼠标触发 Dock 切屏）
- [ ] 期望：猫出现在右侧屏幕并贴在 Dock 上方（而不是跑回左侧主屏）
- [ ] 期望：猫的活动范围以右侧屏幕的 `visibleFrame` 为准（不跨屏）

### 4) 双屏，主屏在右，Dock 在左侧屏幕底部（全局 x 为负）

- [ ] 设置显示器排列：副屏在左，主屏在右（左侧屏幕全局 x 为负）
- [ ] 将 Dock 移动/切换到左侧屏幕底部
- [ ] 期望：猫出现在左侧屏幕并贴在 Dock 上方（不被错误地“夹回”右侧屏幕）
- [ ] 期望：猫在左侧屏幕的水平坐标允许为负（表现为猫确实在左屏可见区域内活动）

### 5) 两块屏幕缩放不同（例如 2x + 1x）

- [ ] 设置：一块 Retina（常见 2x 缩放）+ 一块外接 1x（或“更多空间/默认”导致缩放不同）
- [ ] 将 Dock 放在缩放为 2x 的屏幕底部，观察猫贴边与活动范围
- [ ] 将 Dock 放在缩放为 1x 的屏幕底部，观察猫贴边与活动范围
- [ ] 期望：两种缩放下猫都贴在正确屏幕的 Dock 上方，且活动范围与可见区域一致（不出现明显“偏移一截”）

### 6) Dock 在左侧，猫不进入 Dock 占用区域

- [ ] 设置：Dock 位置 = 左侧（可选：自动隐藏开/关各验证一次）
- [ ] 期望：猫采用 floor mode，仍沿屏幕可见区域底边活动（不做左侧竖向贴边）
- [ ] 期望：猫的活动范围以 `visibleFrame` 为准，**不会进入左侧 Dock 占用区域**

### 7) Dock 在右侧，猫不进入 Dock 占用区域

- [ ] 设置：Dock 位置 = 右侧（可选：自动隐藏开/关各验证一次）
- [ ] 期望：猫采用 floor mode，仍沿屏幕可见区域底边活动（不做右侧竖向贴边）
- [ ] 期望：猫的活动范围以 `visibleFrame` 为准，**不会进入右侧 Dock 占用区域**
