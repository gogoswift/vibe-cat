# Dock Layout Regression Checklist

本清单用于人工回归验证 **Vibe Cat 的 Dock 相关布局逻辑** 在多屏幕、不同缩放、Dock 自动隐藏、Dock 位于左/右侧等场景下的表现是否符合预期。

支持边界（本轮应满足）：

- 多屏支持：猫会锚定到 **Dock 所在屏幕**，而不是固定主屏。
- 自动隐藏 Dock：Dock 显示/隐藏后，猫会在短时间内回到 **同一块屏幕的 Dock 上方**（避免停留在旧位置）。
- 左/右 Dock：Dock 位于左/右侧时，猫仍只沿屏幕 **底边** 活动（不沿左/右边竖着贴边走），并且不会进入 Dock 图标所在的竖向占用区域。

不在本清单覆盖范围：

- 动画状态/行为映射（走路、奔跑、睡觉等）本身的正确性。
- 气泡/权限提示/子代理迷你猫的业务逻辑。

---

## 预置条件

- macOS 系统可进入“系统设置 / 桌面与 Dock（Desktop & Dock）”。
- 已启动应用（任选其一）：
  - 运行已构建的 `VibeCat.app`
  - 或在仓库根目录执行 `cargo run -- cat` 启动猫窗口
- 若涉及双屏：
  - 可在“系统设置 / 显示器（Displays）”调整屏幕排列（左右位置）与缩放（2x/1x）。

---

## 统一判据（时间/容差）

除非某条用例另有说明，本清单使用以下可执行判据：

- **响应时延**：当你完成 Dock 设置变更，或触发 Dock 显示/隐藏后，猫应在 **0.5 秒内**移动到正确屏幕的正确位置并稳定；若 macOS 动画/系统负载导致延迟，可容忍最多 **1.0 秒**。
- **正确屏幕**：猫应出现在 **当前实际显示 Dock 的那块屏幕** 上；持续观察 **10 秒**，猫不应跨到另一块屏幕。
- **不侵入 Dock 区域**：
  - Dock 在底部：猫应位于 Dock 上方，猫窗口不应覆盖 Dock 图标区域。
  - Dock 在左/右：猫不应与 Dock 竖条重叠（不应遮挡任何 Dock 图标）。

---

## 如何确认 Dock 已经切到目标屏（双屏必做）

1. **让 Dock 处于可见状态**：
   - 若开启了自动隐藏：将鼠标移到目标屏幕对应的边缘（底部/左侧/右侧）并停留约 1 秒，直到 Dock 出现。
   - 若未开启自动隐藏：直接观察哪一块屏幕边缘正在显示 Dock。
2. **成功判据（必须满足）**：
   - Dock 图标条出现在你期望的目标屏幕边缘；
   - 另一块屏幕对应边缘此时不显示 Dock（或在自动隐藏开启时，移过去不会“抢回”Dock）。
3. **失败时备用操作**（从上往下依次尝试，避免“凭感觉”）：
   - 暂时关闭“自动隐藏和显示 Dock”，重复第 1 步让 Dock 在目标屏保持可见，再继续测试。
   - 将鼠标移到目标屏幕边缘并停留更久（约 2 秒），确认 Dock 已在目标屏显示后再开始观察猫的位置。
   - 若 Dock 始终无法切到目标屏：记录当前系统设置与复现路径（见文末“结果记录”），并在备注中标明“Dock 无法切屏”（该现象可能与系统设置/空间配置有关）。

## 回归项（逐条勾选）

### 1) 单屏，Dock 在底部，自动隐藏关闭

- [ ] 设置：Dock 位置 = 底部；“自动隐藏和显示 Dock” = 关闭
- [ ] 期望：猫位于 Dock 上方，且猫窗口不覆盖 Dock 图标（允许猫与 Dock 顶边之间存在少量留白，约 <= 10px）
- [ ] 期望：持续观察 10 秒，猫始终在当前屏幕内活动（不跨到其他屏幕/不跑到屏幕外）

### 2) 单屏，Dock 在底部，自动隐藏开启/关闭切换

- [ ] 初始设置：Dock 位置 = 底部；“自动隐藏和显示 Dock” = 关闭；确认猫已贴边
- [ ] 切换：打开“自动隐藏和显示 Dock”，将鼠标移开 Dock 触发隐藏
- [ ] 期望：Dock 隐藏后 0.5 秒内，猫仍停留在屏幕底边附近，且不覆盖（已隐藏的）Dock 原位置区域造成明显“悬空”
- [ ] 切换：将鼠标移到底部呼出 Dock（或触发显示），再关闭自动隐藏
- [ ] 期望：Dock 显示后 0.5 秒内，猫回到 Dock 上方且不遮挡 Dock 图标；关闭自动隐藏后仍满足该判据

### 3) 双屏，主屏在左，Dock 在右侧屏幕底部

- [ ] 设置显示器排列：主屏在左，副屏在右
- [ ] 操作：按“如何确认 Dock 已经切到目标屏”先确认 Dock 已在 **右侧屏幕底部** 可见
- [ ] 期望：Dock 出现在右侧屏幕后 1.0 秒内，猫也出现在右侧屏幕，且位于 Dock 上方（不遮挡 Dock 图标）
- [ ] 期望：持续观察 10 秒，猫不跨到左侧屏幕

### 4) 双屏，主屏在右，Dock 在左侧屏幕底部（左侧屏幕在“空间排列”中位于左边）

- [ ] 设置显示器排列：副屏在左，主屏在右
- [ ] 操作：按“如何确认 Dock 已经切到目标屏”先确认 Dock 已在 **左侧屏幕底部** 可见
- [ ] 期望：Dock 出现在左侧屏幕后 1.0 秒内，猫也出现在左侧屏幕，且位于 Dock 上方（不遮挡 Dock 图标）
- [ ] 期望：持续观察 10 秒，猫不应“被挤回”右侧屏幕

### 5) 两块屏幕缩放不同（例如 2x + 1x）

- [ ] 设置：一块 Retina（常见 2x 缩放）+ 一块外接 1x（或“更多空间/默认”导致缩放不同）
- [ ] 操作：将 Dock 切到缩放为 2x 的屏幕并保持可见（参考“如何确认 Dock 已经切到目标屏”）
- [ ] 期望：猫在 1.0 秒内出现在该屏幕并位于 Dock 上方（不遮挡 Dock 图标）
- [ ] 操作：将 Dock 切到缩放为 1x 的屏幕并保持可见
- [ ] 期望：猫在 1.0 秒内出现在该屏幕并位于 Dock 上方（不遮挡 Dock 图标）
- [ ] 期望：两种缩放下，猫与 Dock 的相对位置不出现明显异常（例如猫与 Dock 之间出现 > 20px 的大空隙，或猫压住 Dock 图标）

### 6) Dock 在左侧，猫不进入 Dock 占用区域

- [ ] 设置：Dock 位置 = 左侧（可选：自动隐藏开/关各验证一次）
- [ ] 期望：猫仍只在屏幕底边附近活动（不会沿左侧竖向“贴边走”）
- [ ] 期望：持续观察 10 秒，猫不与左侧 Dock 竖条重叠（不遮挡任何 Dock 图标）

### 7) Dock 在右侧，猫不进入 Dock 占用区域

- [ ] 设置：Dock 位置 = 右侧（可选：自动隐藏开/关各验证一次）
- [ ] 期望：猫仍只在屏幕底边附近活动（不会沿右侧竖向“贴边走”）
- [ ] 期望：持续观察 10 秒，猫不与右侧 Dock 竖条重叠（不遮挡任何 Dock 图标）

### 8) 双屏垂直排列：副屏在主屏上方，Dock 在上方副屏底部

- [ ] 设置显示器排列：副屏在上，主屏在下（上下排列而非左右排列）
- [ ] 操作：按“如何确认 Dock 已经切到目标屏”先确认 Dock 已在 **上方副屏底部** 可见
- [ ] 期望：Dock 出现在上方副屏后 1.0 秒内，猫也出现在上方副屏，并位于 Dock 上方（不遮挡 Dock 图标）
- [ ] 期望：持续观察 10 秒，猫不跨到下方主屏

### 9) 双屏垂直排列：副屏在主屏下方，Dock 在下方副屏底部

- [ ] 设置显示器排列：主屏在上，副屏在下（上下排列而非左右排列）
- [ ] 操作：按“如何确认 Dock 已经切到目标屏”先确认 Dock 已在 **下方副屏底部** 可见
- [ ] 期望：Dock 出现在下方副屏后 1.0 秒内，猫也出现在下方副屏，并位于 Dock 上方（不遮挡 Dock 图标）
- [ ] 期望：持续观察 10 秒，猫不跨到上方主屏

### 10) 双屏，side Dock + 自动隐藏 + Dock 在副屏（非主屏）

- [ ] 设置显示器排列：两块屏幕任意（左右或上下均可），但确保 Dock 最终落在副屏
- [ ] 设置：Dock 位置 = 左侧或右侧；“自动隐藏和显示 Dock” = 开启
- [ ] 操作：按“如何确认 Dock 已经切到目标屏”先确认 Dock 已在 **副屏侧边** 可见
- [ ] 操作：将鼠标移开 Dock 触发隐藏，等待 1 秒（确保 Dock 处于隐藏态）
- [ ] 期望：Dock 隐藏态下猫仍保持在副屏（不应回退/跳回主屏）
- [ ] 操作：将鼠标移到副屏对应边缘呼出 Dock，再移开触发隐藏（重复 2 次）
- [ ] 期望：多次显示/隐藏过程中，猫始终保持锚定在副屏，不发生“锚屏回退到主屏”的跳变
